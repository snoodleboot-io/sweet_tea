## Purpose

This workflow provides comprehensive guidance for design rest, graphql, and grpc apis. It ensures teams follow best practices, consider trade-offs, and make informed decisions.

## When to Use This Workflow

Use this workflow when:
- Starting a new initiative in this domain
- Making architectural decisions
- Redesigning existing systems
- Optimizing current implementations
- Onboarding teams to the domain

## Workflow Steps

### Phase 1: Planning & Analysis
- Define goals and constraints
- Understand business requirements
- Identify technical requirements
- Document assumptions

### Phase 2: Design & Evaluation
- Identify viable options
- Evaluate each option against requirements
- Document pros/cons for each
- Select the best approach
- Design the implementation

### Phase 3: Implementation
- Create implementation plan
- Set up development environment
- Implement the solution
- Follow coding standards
- Maintain documentation

### Phase 4: Testing & Validation
- Develop test strategy
- Write tests
- Execute testing
- Validate against requirements
- Performance testing
- Security testing

### Phase 5: Deployment & Rollout
- Plan deployment strategy
- Deploy to staging
- Conduct UAT
- Deploy to production
- Monitor deployment
- Rollback plan if needed

### Phase 6: Operations & Monitoring
- Monitor system health
- Track metrics
- Handle incidents
- Plan optimization
- Gather feedback
- Continuous improvement

## Best Practices

- Document decisions with clear rationale
- Involve stakeholders early
- Test thoroughly before production
- Plan for failure and rollback
- Monitor and measure outcomes
- Gather feedback and iterate
- Share learnings with the team

## Common Pitfalls to Avoid

- Over-engineering for hypothetical scenarios
- Ignoring operational complexity
- Skipping testing or shortcuts
- Poor documentation
- Not involving the right stakeholders
- Lack of monitoring and observability
- No rollback plan

## Success Criteria

- Requirements are fully met
- Trade-offs are explicit and documented
- Team understands and supports the approach
- Solution is maintainable and scalable
- Performance meets targets
- Security requirements are satisfied
- Monitoring and alerting are in place
- Documentation is complete
- Team is trained and confident
